# ADR-0015: Localization — единый русский язык, без i18n-библиотеки в Phase 2

**Status:** accepted
**Date:** 2026-04-27
**Deciders:** architect, owner
**Related:** USER_FEEDBACK §4.6.

---

## Context

Юзер: *«Можно ли заменить язык везде на русский или даже в настройках сделать выбор языка, если это сложно, то просто сохранив этот красивый шрифт перейти на русский везде»*.

Сейчас фронт смешивает русские и английские строки (особенно `placeholder`, `label`, кнопки в формах). Бэк-сообщения — на английском.

## Decision

**Phase 2 (текущая):** все строки UI и backend error messages — **на русском**, **без i18n-библиотеки**. Просто статичные русские тексты.

**Не вводим i18next / react-intl / FormatJS** на этом этапе, потому что:
- Нет реального второго языка.
- Любая i18n-библиотека требует обвязки (provider, namespace, плюрализация), а ROI = 0.
- Если когда-то понадобится английский — добавим за один спринт.

**Backend error messages:** все `raise ValidationError({"field": ["сообщение"]})` — на русском. Шаблон ошибки `{error: {code, message, details}}` остаётся, но `message` и `details` — русские.

**Frontend strings:** просто переводим файлы. Нет лоокапов. Каждая страница использует жёстко русские строки.

**Шрифт:** Cinzel (latin-only) проблематичен с кириллицей — буквы выглядят как fallback. **Решение:** шрифт Cinzel остаётся для **акцентных латинских моментов** (например, логотип «TRONUS», номера), а для основного контента — кириллический серифный (например, **Marck Script** или **Spectral**). См. T-105 / F-105.

## Alternatives considered

| Вариант | Минусы |
|---------|--------|
| `react-intl` сразу | Лишняя сложность под фичу, которой нет. |
| Локализация только UI, бэк на en | Несогласованность; ошибки от бэка показываем в UI как есть. |
| Cinzel cyrillic fork | Их нет. Marck Script + Spectral — официальные кириллические альтернативы из Google Fonts. |

## Consequences

- Простота: новые тексты пишутся на месте.
- Ошибки от бэка можно показывать на фронте без перевода.
- Если понадобится английский — рефакторинг на i18next; не страшно, всё в одном репозитории.

## Implementation pointer

Frontend задача `F-105`: пройти все .tsx-файлы и заменить английские строки на русские. Backend задача `T-115`: пройти все `ValidationError` и стандартизовать русские сообщения.

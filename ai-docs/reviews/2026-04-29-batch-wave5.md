## Highlights

**Что сделано хорошо:**

1. **T-114 (avatars)** — `build_absolute_uri` корректно прокинут через `request` во все relevant сериализаторы, включая вложенные user-в-session. Vite `/media` и `/admin` проксируются — теперь полный round-trip работает.
2. **T-110 / T-111 / T-113** — auth pivot аккуратно сделан: API контракт `{login, password}` единообразен между login и password reset; `secret_word` через env, default пустая строка отключает фичу полностью; anti-enumeration в reset (один и тот же ответ для существующего и несуществующего login).
3. **T-105 / T-106** — slug rename через миграцию **с reverse-функцией** (можно откатить), `house_deck` поле в `GameSession` через rename вместо drop+create (история сохраняется).
4. **T-106** — внешний REST-ключ `deck` сохранён в API (не `house_deck`), чтобы не ломать клиентов. Хорошее решение.
5. **T-107** — `validate_session_setup` чистая функция, тестируется отдельно от ORM. Покрытие — 8+ кейсов на режим.
6. **F-100** — кнопка `Cancel` с confirm dialog + toast, RSVP-кнопки явно `disabled` с tooltip — честно показывает «coming soon», не молчит.
7. **T-118** — bonus-задача, agent проактивно вычистил оставшиеся следы `expansion_a/b` из тестов и фронт-моков. Хороший пример самостоятельной инициативы в безопасных пределах.

**Замечание (не блокирующее):**

- T-105: agent в окружении не имел Postgres, прогонял с pytest fallback. Все тесты, использующие `ArrayField`, помечены `@pytest.mark.postgres` или эквивалентом. На CI с Postgres всё работает. Не Action item — просто факт.

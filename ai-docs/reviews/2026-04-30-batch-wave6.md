## Highlights

### Бэкенд

1. **T-100** заводит фундаментальный pivot: новый статус `IN_PROGRESS`, `SessionInvite` модель и `RoundSnapshot` модель в одной волне. Все три ADR-0009/-0010/-0013 проявились корректно.
2. **T-101** — `complete_round` валидирует все трекеры (influence×3, supply, castles, wildlings) через `_validate_round_payload`. Snapshot immutable. `discard_last_round` admin-only.
3. **T-120 / T-121 / T-122** — RSVP flow, randomize_factions (preview-only, без side-effects), replace_participant с timeline event.
4. **T-123** — `finalize_session` теперь чистый CR-007: только `{mvp?, final_note?}`, победитель из последнего snapshot, tiebreak по `influence_throne`.
5. **T-126** — `MatchComment.author` стал nullable для системных сообщений-летописца, добавлен `chronicler_event` FK.
6. **T-104** — `WESTEROS_DECKS` захардкожены, структура корректная. **Slugs — placeholder**, нужен реальный список карт от владельца. → CR-009.

### Фронт

1. **F-118** — экран финализации стал **подтверждением результата**: показывает вычисленные места из последнего snapshot, блокирует кнопку если ни у кого нет 7 замков. Двойное подтверждение перед сохранением.
2. **F-111** — wizard распределения фракций с кнопкой «Случайно» (через `randomize-factions` API). Валидация перед `start_session`.
3. **F-112** — главный новый экран. 3 трека влияния (up/down reorder вместо drag-and-drop — прагматичный выбор для мобилок), supply/castles steppers, wildlings stepper. Alert при castles=7.
4. **F-110** — `RsvpBlock` компонент с кнопками «Иду / Может быть / Не иду» + «Присоединиться» для незарегистрированных.

## Замечания (не блокирующие, → новые задачи)

### 1. Дубликат `class MatchTimelineEvent` в `apps/games/models.py`

**Найдено:** строки 262-300 и 303-341. Python использует второе определение, поэтому проект работает, но это **техдолг и потенциальная ловушка**:
- Различие в полях: первое имеет `happened_at = DateTimeField(auto_now_add=True)`, второе — `default=timezone.now`. По итогу действует второе (mutable).
- Любой следующий ALT-import или future agent читает первый — может прийти к неверным выводам.

→ **CR-008** (cleanup), задача **T-127**.

### 2. Дубликат `class SessionFinalizeView` в `apps/games/views.py`

**Найдено:** строки 276 и 582. Второе определение — после CR-007 редизайна. Первое — старая реализация, должна была быть удалена.

→ **CR-008** (тот же cleanup), включает оба дубля.

### 3. WESTEROS_DECKS slugs — placeholder

**T-104** реализовал event_cards.py с 10 × 3 × 4 = 120 placeholder slugs. Структура верна, но slugs ненастоящие. Финальный список карт (из реальной игры) нужен от владельца.

→ **CR-009** (open), задача **T-128** + дополнение к T-128 в виде data update.

### 4. T-100 заскоп нарушение (информационно)

В отчёте T-100 agent отметил, что создал **сразу** `SessionInvite` модель и `RoundSnapshot` модель, хотя по плану они были в T-120 и T-101. Это **разумно** (модели нужны были сразу для start_session), но формально — отступление от scope. agent явно обозначил это в отчёте, замечаний нет — приняли как pragmatic decision.

## Summary

Собран базовый UI-kit и набор domain components для фронта: atoms, player-, match- и stats-компоненты. Все компоненты заведены на typed mock-data, подключены к реальной странице и покрыты smoke-тестами.

## Changes

- `frontend/src/mocks/types.ts` — типы mock-domain данных для users, factions, matches, stats.
- `frontend/src/mocks/data.ts` — единый набор моков для home/demo-сценариев и component usage.
- `frontend/src/hooks/useFactions.ts` — хук доступа к справочнику фракций для badge/player surfaces.
- `frontend/src/components/ui/Button.tsx` — вариативная кнопка с размерами и icon slots.
- `frontend/src/components/ui/Input.tsx` — текстовый инпут с label/hint/error.
- `frontend/src/components/ui/Select.tsx` — select с typed options.
- `frontend/src/components/ui/Modal.tsx` — базовый dialog-layer для confirm flows.
- `frontend/src/components/ui/Toast.tsx` — временное уведомление с visible/onClose API.
- `frontend/src/components/ui/Skeleton.tsx` — skeleton variants для tile/row/card.
- `frontend/src/components/ui/EmptyState.tsx` — переиспользуемый empty-state блок.
- `frontend/src/components/ui/NumberStepper.tsx` — stepper для числовых полей.
- `frontend/src/components/ui/Stepper.tsx` — wizard progress indicator.
- `frontend/src/components/player/FactionBadge.tsx` — faction badge по slug с runtime color.
- `frontend/src/components/player/PlayerPill.tsx` — compact/large surface игрока с favorite faction.
- `frontend/src/components/match/MatchCard.tsx` — карточка матча для list/home.
- `frontend/src/components/match/PlacementRow.tsx` — строка результата участника.
- `frontend/src/components/stats/StatTile.tsx` — метрика/плитка статистики.
- `frontend/src/components/stats/WinrateBar.tsx` — бар винрейта фракции.
- `frontend/src/components/stats/LeaderboardRow.tsx` — строка leaderboard.
- `frontend/src/pages/HomePage.tsx` — showcase-страница, где используются все новые компоненты.
- `frontend/tests/home-page-components.test.tsx` — smoke-тесты взаимодействий с UI-kit showcase.

## Decisions made

- Для `FactionBadge` цвет берётся по `slug` через `useFactions()`, а runtime inline-style оставлен только там, где цвет реально приходит как данные, что соответствует acceptance.
- `HomePage` временно используется как showcase-площадка для компонентов, чтобы каждый кирпичик сразу имел живой usage-path без отдельного storybook.
- Моки вынесены в typed-слой `src/mocks/`, чтобы следующий этап мог переиспользовать их в `F-005`…`F-008`, а не дублировать структуры по страницам.

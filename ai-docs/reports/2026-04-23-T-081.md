## Decisions made

- Used a separate `backend/Dockerfile.prod`, because the active dev workflow must stay on `runserver` with zero ambiguity.
- Put `migrate + collectstatic` into a small entrypoint instead of keeping a long shell command in compose, so the actual runtime command remains the clean `gunicorn ...`.

---

## Deviations from task

- Added `backend/entrypoint.prod.sh`, although it was not listed explicitly in `Files to touch`. This keeps the production container startup readable and avoids embedding multi-step boot logic in compose.
- Restored `frontend/src/build/*` stubs although they were outside the requested file list. Reviewer cleanup removed them, but `vite.config.ts` still references these files for production aliasing, so the draft deploy stack could not build without restoring them.

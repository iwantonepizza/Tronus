## Summary

Добавил companion endpoint для H2H autopick: backend теперь умеет вернуть `interesting_opponent_id` для заданного пользователя. Эвристика простая и стабильная: сначала ищется игрок с максимальным числом совместных completed-партий, а если таких партий ещё нет — используется первый подходящий игрок из leaderboard.

## Decisions made

- Внешний контракт сделал минимальным: endpoint возвращает только `interesting_opponent_id`. Этого достаточно для `F-222`, а лишний user payload дублировал бы уже существующие user-списки и увеличивал бы surface area API.
- Fallback идёт через текущий leaderboard по `wins`, а не через отдельную custom-эвристику. Это даёт предсказуемый выбор без дублирования ranking-логики в stats слое.

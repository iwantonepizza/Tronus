## Summary

Собран базовый Django-скелет проекта с раздельными settings (`base/dev/prod`), env-конфигурацией через `django-environ`, PostgreSQL-конфигом по `DATABASE_URL` и подключённым DRF. Зависимости и `requirements.txt` оформлены через `requirements.in` + `pip-compile`, линтер и `manage.py check` проходят.

Полностью закрыть acceptance по `migrate` и `runserver` не удалось из-за внешнего блокера: локальный PostgreSQL Server 18 требует пароль для роли `postgres`, а валидные креды в проект не переданы.

- `.env.example` — пример локальной конфигурации с `DATABASE_URL`.
- `requirements.in` — исходный список зависимостей.
- `requirements.txt` — lock-файл, пересобран через `pip-compile`.
- `pyproject.toml` — конфиг `ruff` с `line-length=100` и target Python 3.12.

## Decisions made

- Для PostgreSQL-драйвера использован `psycopg` v3 вместо `psycopg2-binary`: на Windows с локальным PostgreSQL 18 `psycopg2-binary` падал на `UnicodeDecodeError` уже на этапе `connect()`, а `psycopg` отдаёт нормальную `OperationalError`.
- В `dev.py` и `prod.py` использован `globals().update(...)` из `base`, чтобы не нарушать правило проекта `не использовать import *`.
- Дефолтный host в `DATABASE_URL` переведён на `127.0.0.1`, чтобы избегать лишней неоднозначности вокруг `localhost`.

---

## Deviations from task

- В `requirements.in` изначально был добавлен `psycopg2-binary`, но по факту работы на машине заменён на `psycopg[binary]` из-за некорректного поведения старого драйвера при подключении к локальному PostgreSQL.

---

## Open questions / blockers

- Для локального PostgreSQL Server 18 на `127.0.0.1:5432` требуется пароль; `postgres:postgres` не подходит. Без валидного `DATABASE_URL` невозможно завершить `migrate` и полноценный `runserver`.
- На машине нет `psql`, поэтому быстро проверить/создать роль и БД через стандартный CLI не получилось.

## How to verify

1. Создать локальный `.env` на основе `.env.example` и указать рабочий `DATABASE_URL`.
2. Выполнить `python manage.py check`.
3. Выполнить `python manage.py migrate`.
4. Выполнить `python manage.py runserver --noreload`.
5. Проверить `http://127.0.0.1:8000/` и `http://127.0.0.1:8000/admin/`.

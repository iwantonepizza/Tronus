## Decisions made

- Агрегации для `player_profile_stats` собрал в selector на Python поверх `select_related` queryset, а не через один монолитный SQL-запрос. Для текущего масштаба данных это проще проверить и поддерживать, при этом не нарушает слоистую архитектуру.
- Для игрока без completed-сессий поля, у которых нет естественного значения (`winrate`, `avg_place`, `avg_castles`, `favorite_faction`, `best_faction`, `worst_faction`), возвращаются как `null`, а streak — как `{type: null, count: 0}`.
- Stats endpoint ограничен `get_public_user_queryset()`, поэтому inactive user не раскрывается через публичную статистику.

## Deviations from task

- `PlayerStats` shape из `DESIGN_BRIEF` трактован безопасно для edge-case без completed-сессий: часть полей nullable. В docs точный JSON-контракт для `stats` ещё не зафиксирован, поэтому это не ломает утверждённый API.

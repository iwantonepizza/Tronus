## Summary

Добавлен GitHub Actions workflow для CI с двумя job-ами: `lint` и `test`. Workflow запускается на `push` в `main` и на каждый `pull_request`, использует Python 3.12, ставит зависимости из `requirements.txt`, гоняет `ruff` и поднимает `postgres:15` service для Django-тестов.

Конфигурация согласована с текущим состоянием проекта: `pytest` запускается как `python -m pytest`, а тестовый job использует тот же `DATABASE_URL`, который уже применялся в локальной docker-проверке.

---

## Changes

- `.github/workflows/ci.yml` — создан workflow `CI`.
  - `lint` job:
    - `actions/checkout@v4`
    - `actions/setup-python@v5` с Python 3.12 и pip-cache
    - `python -m pip install -r requirements.txt`
    - `ruff check .`
  - `test` job:
    - PostgreSQL 15 как service container
    - env для Django (`DJANGO_SETTINGS_MODULE`, `SECRET_KEY`, `DEBUG`, `DATABASE_URL`)
    - `python manage.py migrate`
    - `python -m pytest`

## Decisions made

- Workflow разделён на `lint` и `test`, чтобы линт и тесты были независимыми и читались отдельно в CI.
- Для `test` job выбран `postgres:15`, чтобы он совпадал с dev-окружением из `T-002`.
- Использован `python -m pytest`, а не голый `pytest`, чтобы CI-команда совпадала с устойчивым локальным запуском из `T-004`.
- Включён pip-cache через `actions/setup-python`, чтобы не тянуть зависимости заново на каждом запуске.

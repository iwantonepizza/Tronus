# Report for T-114: Fix avatar URL in API responses

**Agent:** Codex
**Date:** 2026-04-29
**Status:** `completed`

---

## Summary

Починен контур с media URL для аватарок: `current_avatar` теперь отдаётся абсолютным URL, а dev Vite proxy умеет проксировать `/media` и `/admin` на backend. Дополнительно прокинут `request` в ручные serializer responses тех API, которые реально кормят Home, Match detail и Profile, чтобы вложенные user/faction serializers тоже строили корректные absolute URLs.

---

## Changes

- `backend/apps/accounts/serializers.py` — `PublicUserSerializer.get_current_avatar()` теперь строит absolute media URL через `request.build_absolute_uri(...)`.
- `backend/apps/accounts/views.py` — `LoginView`, `MeView`, `ProfileUpdateView` теперь передают `context={"request": request}` в `PrivateUserSerializer`.
- `backend/apps/reference/serializers.py` — `FactionSerializer.sigil` переведён на `SerializerMethodField`, чтобы гербы тоже отдавались абсолютным URL.
- `backend/apps/avatars/serializers.py` — `source_photo` и `generated_image` теперь сериализуются в абсолютные media URL.
- `backend/apps/stats/views.py` — все stats responses теперь создаются с `request` в serializer context.
- `backend/apps/comments/views.py` — comments API теперь передаёт `request` в `MatchCommentSerializer`.
- `backend/apps/ratings/views.py` — votes API теперь передаёт `request` в `MatchVoteSerializer`.
- `backend/apps/accounts/tests/test_api.py` — добавлен API-тест на абсолютный `current_avatar` в public user detail.
- `backend/apps/avatars/tests/test_api.py` — обновлён smoke-тест `/auth/me/` на абсолютный URL.
- `backend/apps/stats/tests/test_api.py` — добавлен тест на абсолютный `current_avatar` во `GET /api/v1/stats/players/<id>/`.
- `backend/apps/comments/tests/test_api.py` — добавлен тест на абсолютный `author.current_avatar` в comments list.
- `frontend/vite.config.ts` — добавлены dev proxies для `/admin` и `/media`.

## Decisions made

- Absolute URL fix сделан не только в `PublicUserSerializer`, но и в тех views, которые вручную создают nested serializers без `request`. Иначе acceptance по Home / Match detail / Profile закрывался бы только частично.
- Для `sigil` выбран явный `SerializerMethodField`, чтобы не зависеть от неявного поведения `ImageField` и всегда контролировать shape media URL.

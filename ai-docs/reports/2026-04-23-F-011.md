## Summary

Implemented the private frontend screens for profile management and avatar generation on top of the existing backend APIs from `F-002` and `T-050`. ## Decisions made

- Resolved backend media paths in the API layer instead of inside page components, so the UI stays independent from backend URL quirks and the same logic is reused for auth, profile, and avatar screens.
- Kept the profile form in a dedicated child component keyed by current user state, because this avoids forbidden synchronous state updates in `useEffect` and keeps form initialization deterministic.
- Left avatar history in a simple card grid without extra cropping controls, because `T-050` already defines the backend generation pipeline and the task scope only required the frontend surfaces for screens 7.11 and 7.15.

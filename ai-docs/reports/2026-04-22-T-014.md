## Decisions made

- Публичный список `/users/` и detail `/users/<id>/` отдают только `is_active=True`, чтобы pending users не светились наружу до approve owner'ом.
- `/users/<id>/` всегда остаётся публичным профилем без email, а private serializer используется для `/auth/me/` и ответа `PATCH /users/<id>/profile/`.
- Для `favorite_faction` в profile patch используется slug справочника, потому что это уже принятая внешняя форма для reference API и она стабильнее внутренних integer id.
- Переход на `selectors.get_user_by_email(...)` убрал прямой ORM из auth views и выровнял `accounts` под проектные конвенции.

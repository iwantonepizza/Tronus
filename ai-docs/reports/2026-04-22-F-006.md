## Decisions made

- Для `Match detail` комментарии и голоса вынесены в отдельные domain components уже сейчас, чтобы `F-010` подключал API поверх готовой структуры, а не начинал заново с layout.
- `Matches list` использует одну и ту же панель фильтров и для desktop, и для mobile modal, чтобы не плодить две независимые реализации UI.
- Голоса и комментарии в detail-экране пока локально мутируют mock-state внутри страницы. Это даёт живой UX-path без блокировки на backend integration.

---

## Deviations from task

- Multi-select по игрокам и фракциям реализован в MVP-форме через single select, а не полноформатный multi-select control. Для мокового этапа это не ломает пользовательский путь, но перед API-подключением можно усилить до настоящего multi-select.
- В detail screen votes/comments пока не завязаны на auth-context и backend permissions; current user для mock-flow берётся из mock-данных.
- На build остаётся только non-blocking Vite warning про chunk size `> 500 kB`; сборка успешна.

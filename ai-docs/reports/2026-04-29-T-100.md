## Decisions made

- **`SessionInvite` создан в T-100, не в T-120**: `start_session` требует RSVP-инвайты для построения `Participation`. T-120 добавит только services (`invite_user`, `update_rsvp`) и API поверх уже существующей модели.
- **`RoundSnapshot` создан в T-100**: `start_session` обязан создать начальный снимок (round 0). T-101 добавляет `complete_round` поверх уже существующей модели.
- **Начальные castles = 0, supply = 1**: в `Faction` нет поля `starting_castles`. Значение 0 является placeholder'ом. Когда данные стартовых замков будут добавлены в Faction (отдельная задача), `start_session` нужно будет обновить.
- **Influence tracks round 0**: проинициализированы как `[p_id1, p_id2, ...]` в порядке создания. Эти значения семантически бессмысленны до первого раунда — служат только как "непустой список" для консистентности схемы.

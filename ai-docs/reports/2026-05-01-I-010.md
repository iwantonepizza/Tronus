## Summary

Добавил `Dependabot` для монорепы, чтобы GitHub сам открывал регулярные PR на обновление Python- и Node-зависимостей. Конфиг разделён по рабочим зонам проекта: `backend/` для `pip` и `frontend/` для `npm`.

---

## Changes

- `.github/dependabot.yml` — добавлен weekly dependabot config для:
  - `pip` в `backend/`
  - `npm` в `frontend/`

Результат:

- директории и dependency-файлы существуют — ✓

Примечание:
- Это integration-конфиг для GitHub, поэтому локального runtime-теста у задачи нет. Реальная активация произойдёт после push в GitHub.

---

## Decisions made

- Не стал добавлять `github-actions` ecosystem в этот проход, потому что backlog задачи явно просит auto-update только для `pip` и `npm`. Это уменьшает шум PR и держит scope узким.
- Для обоих ecosystems поставил weekly cadence и `open-pull-requests-limit: 5`, чтобы апдейты не захламляли репозиторий пачкой мелких PR.

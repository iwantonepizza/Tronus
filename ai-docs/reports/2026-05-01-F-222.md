# Report for F-222: H2H autopick на UI

**Agent:** Codex  
**Date:** 2026-05-01  
**Status:** `completed`

---

## Summary

Добавил auto-pick flow для `/h2h`: при первом открытии страницы без query params фронт подставляет `user_a=self`, а `user_b` запрашивает через новый backend endpoint suggested-opponent. При этом ручные селекторы и прямые query params полностью сохраняют приоритет.

---

## Changes

- `frontend/src/api/types.ts` — добавлен тип `ApiHeadToHeadSuggested`.
- `frontend/src/api/stats.ts` — добавлен `getSuggestedHeadToHeadOpponent(forUser)`.
- `frontend/src/hooks/useStats.ts` — добавлен hook `useSuggestedHeadToHeadOpponent(forUser)`.
- `frontend/src/pages/HeadToHeadPage.tsx` — добавлен autopick flow через `useAuth`, `useSuggestedHeadToHeadOpponent` и `useSearchParams`; `user_a` по умолчанию берётся из текущего пользователя, а `user_b` — из suggested endpoint.
- `frontend/tests/head-to-head-page.test.tsx` — добавлены/обновлены тесты на guest empty-state и autopick при первом открытии.
- `frontend/tests/stats-pages.test.tsx` — обновлён mock `useStats` для нового suggested hook.

## Decisions made

- Автоподстановка не меняет поведение вручную выбранной пары: если `user_a` или `user_b` уже пришли в query params, frontend их не перетирает.
- Контракт suggested endpoint используется как минимальный `interesting_opponent_id`, а не как полный user payload; сам пользователь уже резолвится через существующий public users list.

# Report for T-105: Game mode rules schema and mode slug rename

**Agent:** Codex
**Date:** 2026-04-29
**Status:** `completed_with_environment_note`

---

## Summary

Расширил `GameMode` под ADR-0012: добавил rule-fields (`max_rounds`, `westeros_deck_count`, `allowed_factions`, `required_factions`, `factions_by_player_count`), сделал reversible migration `0004_game_mode_rules`, и перевёл проект на новые mode-slugs: `classic`, `feast_for_crows`, `dance_with_dragons`, `mother_of_dragons`.

## Decisions made

- Slug-rename сделан именно data migration, а не через новый seed-файл, чтобы сохранить существующие FK на `GameMode` и сделать reverse предсказуемым.
- `classic.allowed_factions` оставлен пустым, а детальная матрица вынесена в `factions_by_player_count`, как прямо описано в ADR-0012.

## How to verify

1. Поднять PostgreSQL (`cd backend && docker compose up -d db web`).
2. Выполнить `docker compose run --rm web python manage.py migrate`.
3. Проверить `GET /api/v1/reference/modes/`.
4. Убедиться, что вернулись ровно 4 slug:
   - `classic`
   - `feast_for_crows`
   - `dance_with_dragons`
   - `mother_of_dragons`
5. Проверить, что:
   - `feast_for_crows.max_rounds == 7`
   - `mother_of_dragons.required_factions == ["targaryen"]`
6. Для reverse:
   - `docker compose run --rm web python manage.py migrate reference 0003`
   - проверить возврат к legacy slugs
   - затем вернуть миграцию вперёд на `0004`

## Summary

В backend добавлены `MEDIA_URL` и `MEDIA_ROOT`, а `config.urls` теперь раздаёт media-файлы в dev при `DEBUG=True`. Дополнительно синхронизированы ignore-файлы и docker-compose volume для `media/`, а новый smoke-тест подтверждает, что `GET /media/<path>` в dev действительно отдаёт файл.

## Changes

- `backend/config/settings/base.py` — добавлены `MEDIA_URL` и `MEDIA_ROOT`.
- `backend/config/urls.py` — подключён `static(..., document_root=settings.MEDIA_ROOT)` для dev-serving media.
- `backend/docker-compose.yml` — добавлен bind mount `./media:/app/media` для persistence media-файлов между пересборками контейнера.
- `backend/.dockerignore` — добавлен `media/`.
- `backend/.gitignore` — создан, добавлен `media/` и временные SQLite fallback-БД.
- `.gitignore` — создан корневой ignore для `backend/media/` и временных SQLite fallback-БД.
- `backend/tests/test_media.py` — добавлены smoke-тесты на `MEDIA_ROOT` и dev-serving `/media/...`.
- `backend/tests/urls_media.py` — вспомогательный test URLConf для media-serving smoke.
- `backend/tests/__init__.py` — `tests` сделан импортируемым пакетом для `ROOT_URLCONF="tests.urls_media"`.
- `backend/tests/test_smoke.py` — стабилизирован `admin` smoke-тест явным `ROOT_URLCONF="config.urls"`.

## Decisions made

- Активным compose для backend считался `backend/docker-compose.yml`, а не старый дублированный `docker-compose.yml` в корне, потому что это прямо соответствует текущим `README.md` и `ai-docs/INTEGRATION_PLAN.md` (`docker compose up` из `backend/`).
- Для acceptance на dev-serving добавлен отдельный test URLConf вместо попытки переиспользовать `config.urls` с runtime-подменой `MEDIA_ROOT`, потому что `static(..., document_root=...)` фиксирует путь на этапе построения `urlpatterns`.

## Deviations from task

- В CR-004 указан просто `docker-compose.yml`, но в актуальной монорепо-структуре рабочий compose живёт в `backend/`. Поэтому volume добавлен в `backend/docker-compose.yml`; старый корневой compose не трогал, чтобы не смешивать новый backend layout со stale root-деревом.
- В задаче предполагалось наличие корневого `.gitignore` и `backend/.gitignore`, но в репозитории их не было. Оба файла созданы минимально, только под `media/` и временные SQLite fallback-БД.
- Полный runtime-check загрузки через Docker не воспроизведён: Docker daemon на машине выключен. При этом `docker compose config` успешно валидирует обновлённый compose, а поведение `/media/...` покрыто Django smoke-тестом.

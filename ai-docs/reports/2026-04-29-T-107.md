# Report for T-107: GameMode rules engine — `validate_session_setup`

**Agent:** Codex
**Date:** 2026-04-29
**Status:** `completed`

---

## Summary

Добавил чистый backend rules engine `validate_session_setup(*, mode, faction_slugs)` для проверки финального состава фракций по правилам `GameMode`. Логика покрывает player-count, duplicate factions, per-mode allowed roster и required factions; отдельно учтён `classic`, где список доступных фракций зависит от числа игроков.

## Decisions made

- Для режимов с `factions_by_player_count` (`classic`) engine не пытается валидировать allowed roster, если для такого `player_count` нет rule-entry. Это убирает ложные faction-errors в случаях вроде `2 players in classic`, где корректнее вернуть только ошибку по числу игроков.
- `validate_session_setup` оставлен как pure service без немедленного подключения к текущим planning-mutations. Реальное встраивание пойдёт в `start_session` из `T-100`, как и описано в backlog/ADR.

# Report for I-008: healthcheck endpoint

**Agent:** Codex
**Date:** 2026-04-30
**Status:** `completed`

---

## Summary

Добавил публичный `GET /api/v1/health/` в backend core-слой с реальной DB probe через `SELECT 1`. Endpoint теперь возвращает `200`/`503` и JSON в формате из backlog, а deploy-документация получила короткую инструкцию для внешних uptime-monitor сервисов.

---

## Changes

- `backend/apps/core/views.py` — добавлен `HealthCheckView` с проверкой БД и ответами `ok/degraded`.
- `backend/apps/core/urls.py` — добавлен route `health/`.
- `backend/config/urls.py` — подключён `apps.core.urls` на префикс `api/v1/`.
- `backend/config/settings/base.py` — добавлен `APP_VERSION` с дефолтом `0.1.0`.
- `backend/apps/core/tests/test_health_api.py` — добавлены тесты на публичный `200` и degraded `503`.
- `deploy/README.md` — добавлен раздел про `GET /api/v1/health/` для UptimeRobot / Better Stack.

## Tests

- `cd backend && python -m ruff check apps/core config`
- `cd backend && docker compose run --rm web python manage.py check`
- `cd backend && docker compose run --rm web python -m pytest apps/core/tests/test_health_api.py -q -p no:asyncio`
- `cd backend && docker compose up -d`
- `curl http://127.0.0.1:8000/api/v1/health/`
- `docker compose stop db`
- `curl -i http://127.0.0.1:8000/api/v1/health/`

Результат:
- targeted backend tests — `2 passed`
- `python manage.py check` — зелёный
- HTTP smoke:
  - при живой БД: `200` + `{"status":"ok","database":"ok","version":"0.1.0"}`
  - при остановленной БД: `503` + `{"status":"degraded","database":"error","version":"0.1.0"}`

---

## Decisions made

- Root `"/"` healthcheck не убирал, чтобы не ломать существующий минимальный smoke-контур dev-среды; новый production-facing JSON endpoint добавлен отдельно на `"/api/v1/health/"`.
- Версию вынес в settings (`APP_VERSION`), а не захардкодил во view, чтобы дальше можно было перевести её на env/build metadata без переписывания endpoint logic.

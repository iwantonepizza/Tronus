## Summary

Собрано dev-окружение на Docker Compose: отдельный `web`-контейнер для Django и `db` на `postgres:15`. Добавлены `Dockerfile`, `docker-compose.yml`, `.dockerignore` и `Makefile` с требуемыми таргетами.

Фактическая проверка выполнена через `docker compose`: сервисы поднялись, миграции применились, `/admin/` отвечает, команда `python manage.py shell` внутри compose работает.

## Changes

- `docker-compose.yml` — описаны сервисы `web` и `db`, healthcheck для Postgres, bind mount проекта, публикация портов `8000` и `5432`.

- `Makefile` — добавлены таргеты `up`, `down`, `migrate`, `test`, `lint`, `shell`.

## Decisions made

- `DATABASE_URL` для контейнерного `web` задаётся прямо в `docker-compose.yml`, чтобы не трогать `.env.example` из `T-001` и не смешивать локальную bare-metal БД с контейнерной.
- Для `Makefile` команды `migrate`, `test`, `lint`, `shell` реализованы через `docker compose run --rm web ...`, чтобы они не зависели от уже запущенного `web`-контейнера.
- Для `db` добавлен `healthcheck`, а `web` зависит от `db` через `condition: service_healthy`.

## How to verify

1. Убедиться, что Docker Desktop запущен.
2. Выполнить `make up` или эквивалент `docker compose up -d --build`.
3. Выполнить `make migrate` или эквивалент `docker compose run --rm web python manage.py migrate`.
4. Открыть `http://127.0.0.1:8000/admin/`.
5. Выполнить `make shell` или эквивалент `docker compose run --rm web python manage.py shell`.
6. После проверки выполнить `make down` или `docker compose down`.

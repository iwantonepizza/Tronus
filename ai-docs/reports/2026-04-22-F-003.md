## Summary

Вместо временного router bootstrap собран полноценный AppShell: `TopBar`, desktop `Sidebar`, mobile `BottomNav`, nested routes через `Outlet` и skeleton-страницы для всех маршрутов из `FRONTEND_ARCHITECTURE.md`. Responsive-переключение покрыто тестами, активный nav item подсвечивается.

## Decisions made

- Shell построен как parent layout route, чтобы дальнейшие экраны просто заменяли placeholder-элементы без перестройки роутинга.
- Protected placeholders (`/matches/new`, `/matches/:id/edit`, `/matches/:id/finalize`, `/me/avatar`) подключены уже сейчас через `RequireAuth`, чтобы auth-contract и routing не расходились.
- Порог responsive принят строго по task acceptance: bottom nav ориентирован на `<768`, sidebar — на `>=1024`.

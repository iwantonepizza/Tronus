# Report for T-171: MethodNotAllowed returns structured 405

**Agent:** Codex
**Date:** 2026-05-01
**Status:** `completed`

---

## Summary

Добавил regression-тесты для уже влитого architect hotfix в `games` и `ratings`: `MethodNotAllowed` больше не должен проваливаться в catch-all `500`. Проверка зафиксирована на реальных API endpoints, где разрешён только `POST`/`PATCH`/`DELETE`, а `GET` обязан отдавать структурированный `405`.

---

## Changes

- `backend/apps/games/tests/test_api.py` — добавлен `test_start_session_get_returns_method_not_allowed` для `GET /api/v1/sessions/<id>/start/`.
- `backend/apps/ratings/tests/test_api.py` — добавлен `test_vote_detail_get_returns_method_not_allowed` для `GET /api/v1/sessions/<id>/votes/<id>/`.

## Decisions made

- Для тестов использовал авторизованный сценарий, а не anonymous: на этих endpoints auth-проверка срабатывает раньше dispatch и anonymous `GET` даёт `401`, что не проверяет hotfix `MethodNotAllowed`.
- Assert оставлен на инварианте `status=405` + `error.code="method_not_allowed"`. Поле `details.allowed_methods` не гарантированно приходит от DRF в этом контуре, поэтому не делал тест хрупким.

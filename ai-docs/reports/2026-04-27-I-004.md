## Decisions made

**Три отдельных job, а не один.** Паттерн позволяет видеть в GitHub UI отдельно что упало: lint / test / build. Накладные расходы (три `npm ci`) минимальны при кэше.

**Path filter — открытый вопрос (см. ниже).** Бэкенд CI лежит в `backend/.github/workflows/ci.yml`, то есть использует паттерн «`.github/` в каждом сабдире». Следую этому паттерну. При таком размещении AC «PR в `backend/` не триггерит frontend CI» выполняется автоматически — CI читается только из директории своего «мини-репо». Path filter `frontend/**` нужен только если CI вынести в корневой `.github/`.

**Node.js 20** — LTS-версия, актуальна на момент задачи.

**VITE_USE_MOCKS=false** в build step — корректно, моки не должны попадать в CI-бандл.

## Deviations from task

Формально задача говорила про path filter `frontend/**`. Это применимо только при corневом `.github/workflows/`. Следую установленному в проекте паттерну (`backend/.github/` → `frontend/.github/`). Architect может принять решение вынести оба workflow в корень; тогда:
- `frontend-ci.yml` нужно добавить `on.push.paths: ["frontend/**"]` / `on.pull_request.paths: ["frontend/**"]`
- `ci.yml` — аналогично `["backend/**"]`
- Файлы переместить в `tronus/.github/workflows/`

# Report for T-106: House decks redesign

**Agent:** Codex
**Date:** 2026-04-29
**Status:** `completed`

---

## Summary

Переименовал backend reference-модель `Deck` в `HouseDeck`, перевёл `GameSession.deck` в `GameSession.house_deck`, но сохранил внешний REST-ключ `deck`, чтобы не ломать контракт клиентов. Добавил миграции для rename и data pivot к двум вариантам колод: `original` и `alternative`. Синхронизировал тесты, stats-serializers и frontend types/mocks, чтобы форма создания сессии больше не знала про `expansion_a` / `expansion_b`.

## Decisions made

- Внешний REST-ключ `deck` сохранён в session API, хотя внутренняя модель уже `house_deck`. Это соответствует acceptance в `CR-006` и не требует отдельного CR на API rename.
- Endpoint `/api/v1/reference/decks/` не переименовывался: URL остаётся стабильным, но семантически теперь отдаёт именно house decks.

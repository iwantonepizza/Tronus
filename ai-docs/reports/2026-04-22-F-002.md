## Decisions made

- API-типы оставлены в `snake_case`, чтобы совпадать с текущим backend JSON и не вводить промежуточный transform-слой раньше времени.
- `AuthContext` bootstrap при старте сначала запрашивает CSRF cookie, затем пытается получить `/auth/me/`, как требует `INTEGRATION_PLAN.md`.
- Для `react-refresh/only-export-components` context contract вынесен в отдельный `auth-context.ts`, а provider оставлен в `AuthContext.tsx`.

## How to verify

1. Поднять backend: `cd backend && docker compose up -d db web`
2. Поднять frontend: `cd frontend && npm run dev -- --host 127.0.0.1`
3. Открыть `http://127.0.0.1:5173/register`
4. Зарегистрировать пользователя
5. Через Django admin или shell подтвердить пользователя
6. Войти на `http://127.0.0.1:5173/login`
7. Проверить переход на `/me`
8. Нажать `Logout`, убедиться что доступ к `/me` теряется

**Agent:** codex

## Summary

Added a dedicated GitHub Actions workflow for the frontend with `npm ci`, `lint`, `test`, and `build`, scoped to `frontend/**` changes. Also repaired the backend workflow after the monorepo cleanup so it points to real `backend/` paths instead of deleted root-level files.

During verification I found that the full frontend test suite was flaky under a full parallel run: the same files passed individually, but the whole suite hit default timeout limits. I stabilized the test harness by increasing Testing Library async waits and the Vitest test timeout so the new CI workflow is green in the current repo state.

---

## Changes

- `.github/workflows/frontend-ci.yml` - new frontend workflow with Node 22, npm cache, and `lint/test/build`.
- `.github/workflows/ci.yml` - updated backend workflow to use `backend/**` path filters and real monorepo paths (`backend/requirements.txt`, `backend/manage.py`).
- `frontend/tests/setup.ts` - increased Testing Library async timeout to reduce false negatives under full-suite runs.
- `frontend/vite.config.ts` - raised Vitest `testTimeout` to keep the full suite stable in CI.

## Decisions made

- Used a dedicated `frontend-ci.yml` instead of mixing frontend and backend jobs into one file, because the repo now has clearly separated app roots and separate path filters are easier to reason about.
- Repaired the existing backend workflow as part of the same pass, because after repo cleanup the old file was stale and pointed at deleted root-level Python entrypoints.
- Stabilized test timeouts in the shared frontend test harness instead of weakening or skipping individual tests.

## Deviations from task

- Touched `.github/workflows/ci.yml` in addition to the new frontend workflow. This was necessary to keep CI coherent after the repository cleanup moved all Python entrypoints into `backend/`.
- Touched `frontend/tests/setup.ts` and `frontend/vite.config.ts` because the local acceptance run exposed flaky test timing under the full suite, which would have made the new CI workflow unreliable.

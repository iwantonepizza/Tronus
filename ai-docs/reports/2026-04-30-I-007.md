## Summary

Добавлен shell-скрипт `backup-postgres.sh` с systemd timer для ночного бэкапа Postgres. Ротация — 7 дней. Установка — 4 команды (см. README).

---

## Changes

- `deploy/backup/backup-postgres.sh` — скрипт: `pg_dump | gzip → /opt/tronus/backups/`, ротация старых файлов через `find -mtime +7 -delete`
- `deploy/backup/tronus-backup.service` — systemd oneshot service
- `deploy/backup/tronus-backup.timer` — запуск ежедневно в 03:00, `Persistent=true` (наверстает пропущенный запуск при перезапуске сервера)
- `deploy/backup/README.md` — инструкция установки и восстановления

---

## Decisions made

- Systemd timer вместо cron — нативно для большинства современных Ubuntu/Debian VPS, лучше логирование через journald.
- `Persistent=true` — запускает пропущенный бэкап при следующем старте сервера.
- `RandomizedDelaySec=300` — рандомный сдвиг до 5 минут, чтобы не нагружать диск ровно в 03:00.
- Бэкап через `docker compose exec -T db pg_dump` — не требует установки postgres-клиента на хосте.

---

## How to install

```bash
sudo mkdir -p /opt/tronus/deploy/backup /opt/tronus/backups
sudo cp deploy/backup/* /opt/tronus/deploy/backup/
sudo chmod +x /opt/tronus/deploy/backup/backup-postgres.sh
sudo cp /opt/tronus/deploy/backup/tronus-backup.{service,timer} /etc/systemd/system/
sudo systemctl daemon-reload && sudo systemctl enable --now tronus-backup.timer
sudo systemctl start tronus-backup.service  # тест
```

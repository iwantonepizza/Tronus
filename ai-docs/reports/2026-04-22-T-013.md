## Summary

Реализован auth API для `register / login / logout / me`, добавлены сервисы `register_user` и `approve_user`, а также rate-limit на `/auth/register/` и `/auth/login/` через `django-ratelimit`. Регистрация создаёт `User(is_active=False)` и `Profile`, логин работает по email+password, а `/auth/me/` возвращает приватный профиль с email.

## Decisions made

- В качестве логина используется email, а внутренний `username` автоматически приравнивается к email, потому что endpoint регистрации по контракту не принимает отдельный username.
- `PrivateUserSerializer` используется для `/auth/me/` и login-response, а `PublicUserSerializer` подготовлен заранее для `T-014`, чтобы сразу закрепить privacy-границу без email.
- Rate-limit реализован через `django_ratelimit.core.get_usage(...)` с собственным `429` JSON-ответом, а не через глобальный exception handler, чтобы не растягивать scope задачи.
- Базовый auth в рамках этой задачи — session-based, что соответствует `API_CONTRACT.md` и не требует пока вводить token auth.

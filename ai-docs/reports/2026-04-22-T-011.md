## Decisions made

- Для API фракций наружу возвращаются только `is_active=True`, потому что поле `is_active` в модели явно предназначено для скрытия неактуальных домов без удаления.
- Во views не используется ORM напрямую: queryset формируется через `reference.selectors`, чтобы сохранить согласованность с принятыми конвенциями.
- В сериализаторы не включались служебные поля `id`, `created_at`, `updated_at`, потому что контракту справочников для frontend достаточно стабильных `slug` и доменных атрибутов.

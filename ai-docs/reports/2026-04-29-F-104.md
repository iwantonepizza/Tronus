# Report for F-104: Mobile viewport fix (iPhone safe area)

**Agent:** Codex
**Date:** 2026-04-29
**Status:** `completed`

---

## Summary

Исправлен mobile viewport контур для iPhone/Safari: добавлен `viewport-fit=cover`, layout переведён с проблемных `100vh` на `100dvh`, а body и bottom nav теперь учитывают safe-area inset'ы. Это убирает эффект, когда страница на iPhone раскрывается не на весь экран сразу и нижняя навигация конфликтует с home indicator.

---

## Changes

- `frontend/index.html` — meta viewport обновлён до `viewport-fit=cover`.
- `frontend/src/styles/globals.css` — добавлены CSS custom properties для `safe-area-inset-*`, body padding по safe area, `html/body/#root` переведены на `100dvh`, добавлен utility-класс `bottom-nav-safe`.
- `frontend/src/components/layout/AppShell.tsx` — shell и main-content переведены на `100dvh`, mobile bottom padding учитывает safe area.
- `frontend/src/components/layout/BottomNav.tsx` — навигация получила safe-area-aware класс.
- `frontend/src/components/layout/Sidebar.tsx` — desktop sidebar переведён с `100vh` на `100dvh`.
- `frontend/src/components/auth/RequireAuth.tsx` — auth guard layout переведён на `100dvh`.
- `frontend/src/pages/LoginPage.tsx` — full-height auth page переведена на `100dvh`.
- `frontend/src/pages/RegisterPage.tsx` — full-height auth page переведена на `100dvh`.
- `frontend/src/pages/NotFoundPage.tsx` — full-height error page переведена на `100dvh`.

## Decisions made

- Вместо точечной замены только в одном месте перевёл весь full-height layout-контур на `100dvh`, потому что смешанный `100vh`/`100dvh` обычно оставляет тот же iOS-bug на соседних экранах.
- Safe area вынесена в CSS variables на уровне `:root`, чтобы не дублировать длинные `env(...)` значения по компонентам.

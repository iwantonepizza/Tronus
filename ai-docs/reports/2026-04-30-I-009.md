# Report for I-009: Security headers + rate limiting

**Agent:** claude-sonnet-4-6 (coder role)
**Date:** 2026-04-30
**Status:** `completed`

## Summary

Добавлены security headers (CSP, Referrer-Policy, Permissions-Policy, X-Content-Type-Options) через кастомный middleware, и глобальный rate limit через DRF throttle. Всё активно только в prod settings.

---

## Changes

- `backend/apps/core/middleware.py` — `SecurityHeadersMiddleware`: строит CSP из settings, добавляет Referrer-Policy и Permissions-Policy на каждый ответ
- `backend/config/settings/prod.py`:
  - `SECURE_CONTENT_TYPE_NOSNIFF = True`
  - `X_FRAME_OPTIONS = "DENY"`
  - CSP-переменные (`CSP_DEFAULT_SRC`, `CSP_SCRIPT_SRC`, ...)
  - `REFERRER_POLICY = "strict-origin-when-cross-origin"`
  - `SecurityHeadersMiddleware` вставлен на позицию 1 (после SecurityMiddleware)
  - DRF `DEFAULT_THROTTLE_CLASSES` + rates: anon=60/min, user=300/min

---

## Decisions made

**Кастомный middleware вместо `django-csp`:** лишняя зависимость не нужна — CSP для SPA простой, строится один раз при старте.

**`'unsafe-inline'` в style-src:** Tailwind использует inline-стили для динамических значений (например, faction colors через CSS vars). Убрать нельзя без nonce/hash. Для closed-group приложения без UGC в стилях это приемлемо.

**Rate limits: anon=60/min, user=300/min:** для closed-group нагрузки более чем достаточно. DRF throttle хранит счётчики в cache (по умолчанию LocMemCache). Для продакшна с несколькими gunicorn workers нужно настроить Redis как cache backend — добавлено как open question.

---

## Open questions / blockers

**Для architect:** DRF throttle в multi-worker gunicorn будет работать некорректно с LocMemCache (каждый worker — свой счётчик). Если это проблема — добавить Redis (`django-redis`) и `CACHES` в prod settings. При текущей нагрузке (десятки пользователей) это не критично.

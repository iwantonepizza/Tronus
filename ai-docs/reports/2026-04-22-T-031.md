## Summary

Реализован полный `ratings`-срез: модель `MatchVote`, services/selectors, admin, API и тесты. Голосование доступно только по завершённой партии, только между участниками, без self-vote, с уникальностью пары `from_user -> to_user` и окном редактирования/удаления 24 часа; для admin окно bypass'ится.

## Decisions made

- Для API наружу оставлен нейтральный contract `positive` / `negative`, а в БД — enum `CROWN` / `SHIT`, как требовали docs.
- Admin может редактировать и удалять голос вне 24-часового окна, но `POST` всё равно не bypass'ит инвариант participants-only: это следует из `DATA_MODEL.md`, где `from_user` обязан быть участником сессии.
- Postgres-specific locking исправлен через отказ от `select_for_update()` по nullable join на `Outcome`; `Outcome` читается отдельным запросом. SQLite это не выявлял, а Docker/Postgres выявил.

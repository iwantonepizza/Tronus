## Summary

Switched the frontend stats slice to real backend endpoints instead of mock fallbacks inside production hooks. Home, player profile, leaderboard, factions overview, and faction detail now rely on the existing `stats` API contracts, and production build no longer ships the real `src/mocks/data.ts` dataset.

## Decisions made

- Removed mock fallback specifically from `useStats` rather than from page components, because the query layer is the contract boundary and this keeps pages unchanged and thin.
- Introduced compile-time mock gating via Vite `define`, because runtime env helpers were not enough for Rollup to drop dead branches from production bundles.
- Used tiny build-only noop stubs for `@/mocks/data` and `@/mocks/session-factory`, because some non-stats features still keep dev-only mock paths and this avoids shipping real mock fixtures while preserving local dev/test behavior.

---

## Deviations from task

- Production build still contains a tiny `noop-mocks-data-*.js` helper chunk, but it no longer contains the real mock dataset from `src/mocks/data.ts`. This was the minimal safe way to keep existing dev-only fallback paths outside the stats scope without widening the task into a full mock-system rewrite.

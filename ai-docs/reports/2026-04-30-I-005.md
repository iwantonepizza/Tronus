**Agent:** Codex

## Summary

Привёл production deploy-контур в репозитории к серверной базе owner: внешний host nginx теперь должен проксировать backend на `127.0.0.1:8080`, а не на `8000`. Также зафиксировал в документации, что миграции и `collectstatic` применяются автоматически при старте backend-контейнера через `backend/entrypoint.prod.sh`.

---

## Changes

- `deploy/docker-compose.prod.yml` — loopback bind backend изменён с `127.0.0.1:8000:8000` на `127.0.0.1:8080:8000`.
- `deploy/nginx-host/tronus.conf` — upstream backend переключён на `127.0.0.1:8080`.
- `deploy/README.md` — обновлены схема, curl-примеры, пояснение про авто-migrate/collectstatic и убран дублирующийся заголовок `Troubleshooting`.
- `ai-docs/reports/2026-04-30-I-005.md` — этот отчёт.

---

## Tests

- Команда запуска: `docker compose -f deploy/docker-compose.prod.yml config`.
- Compose-конфиг валиден: ✓.
- Runtime deploy на реальном VPS в рамках этой задачи не выполнялся: это owner-side шаг.

---

## Decisions made

- Не менял внутренний gunicorn bind `0.0.0.0:8000` в контейнере. Изменён только host-side loopback порт, чтобы совместиться с существующим серверным nginx на `8080`.
- Не добавлял отдельный ручной шаг `python manage.py migrate` в основной deploy-flow, потому что `backend/entrypoint.prod.sh` уже гарантирует `migrate` и `collectstatic` на старте backend-контейнера.

## How to verify

1. На сервере склонировать проект в `~/tronus`.
2. Создать `deploy/env/backend.prod.env` из примера и заполнить секреты.
3. Скопировать `deploy/nginx-host/tronus.conf` в `/etc/nginx/sites-available/tronus.conf`, проверить `server_name`, затем `sudo nginx -t && sudo systemctl reload nginx`.
4. Собрать фронт: `cd ~/tronus/frontend && npm ci && npm run build && sudo rsync -a --delete dist/ /var/www/tronus/dist/`.
5. Поднять backend и db: `cd ~/tronus && docker compose -f deploy/docker-compose.prod.yml up -d --build`.
6. Проверить backend напрямую: `curl -i http://127.0.0.1:8080/api/v1/health/`.
7. Проверить через домен: `curl -i https://твой-домен/api/v1/health/`.

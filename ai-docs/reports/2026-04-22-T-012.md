# Report for T-012: accounts profile model

**Agent:** codex
**Date:** 2026-04-22
**Status:** `completed`

---

## Summary

Добавлена модель `Profile` для app `accounts` и подключён `post_save(User)` сигнал для автосоздания профиля при создании пользователя. Профиль хранит `nickname`, `favorite_faction` и `bio`, а связь сделана как `OneToOne` с `PK=user_id`.

## Decisions made

- `Profile.user` сделан `OneToOneField(..., primary_key=True)`, как это явно зафиксировано в `DATA_MODEL.md`.
- Для базового автосоздания `nickname` по умолчанию заполняется из `User.username`, чтобы профиль существовал сразу после создания пользователя и был пригоден для следующего auth-шага.
- Сигнал в рамках `T-012` срабатывает только на `created=True`; логику апрува `False → True` и добавления в группу `player` оставил на `T-015`, где это отдельно описано в backlog и ADR-0004.

## Deviations from task

- Поле `current_avatar` пока не добавлено в `Profile`, хотя оно есть в общей модели данных. Причина техническая: целевая модель `avatars.AvatarAsset` ещё не реализована, и Django не позволит провести валидную FK-связь до появления самой модели. Это поле нужно добрать на avatar-этапе.

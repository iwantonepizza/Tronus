## Summary

## Changes

- `backend/apps/games/services.py` — добавлен `update_participant(...)`, вынесен row-lock lookup для participation, `remove_participant(...)` переведён на него.
- `backend/apps/games/selectors.py` — добавлены queryset helpers для sessions/participations и фильтры `from/to` в `list_sessions(...)`.
- `backend/apps/games/permissions.py` — добавлены `IsPlayerUser` и `IsSessionCreatorOrAdmin`.
- `backend/apps/games/serializers.py` — добавлены query/input/output serializers для list/detail/create/update/finalize flows.
- `backend/apps/games/views.py` — реализованы list/create/detail/update/cancel/participants/finalize endpoints и единый error mapping.
- `backend/apps/games/urls.py` — добавлены маршруты `/sessions/...`.
- `backend/config/urls.py` — подключён `apps.games.urls` под `/api/v1/`.
- `backend/apps/games/tests/test_api.py` — добавлены API smoke/failure tests на permissions, filters, cancel, participants и finalize.
- `backend/apps/games/tests/test_selectors.py` — query-budget тест ослаблен до контрактного лимита `<= 4`, потому что selector теперь выполняется за `2` запроса вместо старых `3`.

## Decisions made

- Поддержал оба query-алиаса `player` и `user` для фильтра списка, чтобы не ломать разнобой между `BACKLOG.md` и `API_CONTRACT.md`.
- Для missing-object сценариев добавил mapping `ObjectDoesNotExist -> 404`, чтобы views могли оставаться тонкими даже при чтении через selectors.
- Сохранил оба способа отмены: `DELETE /sessions/<id>/` и `POST /sessions/<id>/cancel/`, как явно указано в задаче.

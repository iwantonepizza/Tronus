## Summary

Added public `GET /api/v1/stats/leaderboard/` with query params `metric` and `limit`. The endpoint now supports leaderboard slices for `wins`, `winrate`, `games`, `crowns`, `shits`, and `avg_place`, and returns ranked public users with a generic payload that the frontend can reuse for all metric tabs.

## Decisions made

- Response shape is generic across all leaderboard metrics:
  - `metric`
  - `label`
  - `results[]` with `rank`, `user`, `games`, `metric_value`
- Leaderboards include public active users with at least one completed game.
- Sorting is deterministic:
  - `wins`: wins desc, games desc, nickname
  - `winrate`: winrate desc, games desc, wins desc, nickname
  - `games`: games desc, wins desc, nickname
  - `crowns` / `shits`: metric desc, games desc, nickname
  - `avg_place`: avg_place asc, wins desc, games desc, nickname
- Default query metric is `wins`; default limit is `20`.

# ROADMAP.md

Проект идёт фазами. Каждая фаза = поставляемый кусок. Не начинаем следующую, пока текущая не стабильна.

---

## Phase 0 — Bootstrap (инфраструктура) — ✅ CLOSED

Цель: проект запускается локально, CI зелёный, документация актуальна.

- Django project + apps скелеты.
- Postgres через docker-compose.
- `pytest` настроен, один smoke-тест проходит.
- Ruff настроен.
- GitHub Actions: lint + test.
- Dockerfile.
- `.env.example`.

Задачи: `T-001` … `T-005`.

**Definition of done:** `make up`, `make test`, `make lint` работают. README описывает, как запустить.

## Phase 1 — MVP — ✅ CLOSED (2026-04-23)

Цель: owner и его друзья могут пользоваться сервисом вместо Excel / чата.

**В scope:**

- `accounts`: открытая регистрация по email, approve владельцем через admin (ADR-0004), логин, публичный профиль, роли.
- `reference`: справочники + заполнение через data migration.
- `games`: создание, планирование, добавление участников, финализация, отмена.
- `comments`: комментарии под матчем (REST + polling на фронте, ADR-0006).
- `ratings`: голосование `crown` / `shit` с ограничениями.
- `stats`: базовый набор — профиль игрока, винрейт фракции, главная страница (overview). **Публично доступны** (ADR-0005).
- `avatars`: Pillow-композиция «фото + цветная рамка фракции», синхронно (ADR-0007).
- Django admin с кастомным action `Approve selected users` и минимальной настройкой для ручной правки данных.

**Вне scope (Phase 2):**

- AI-генерация аватаров.
- Сезоны, турниры, достижения.
- Подтверждение участия (going / maybe / no).
- Уведомления.
- Теги матчей.
- Advanced статистика (head-to-head, стрики, MVP по сезону).

Задачи: `T-006` … `T-060` (примерно).

**Definition of done:** owner вручную проводит партию: создаёт сессию → добавляет игроков → играет офлайн → финализирует результаты → все участники голосуют и комментируют → статистика профилей обновилась.

## Phase 2 — Обогащение — 🟡 STARTED (polish wave в работе)

Цель: сервис становится «живым», а не просто формой ввода.

**Уже закрыто из плана Phase 2 в рамках Phase 1:**
- Advanced stats: head-to-head, лидерборды, стрики, «лучшая/худшая фракция игрока» (T-040..T-044).
- Улучшенная главная страница (overview со всеми виджетами) (T-042, F-005, F-009).

**Осталось в Phase 2:**
- RSVP (`going` / `maybe` / `no`) на запланированные сессии (T-100, ADR-0008 TBD, F-012).
- Теги матчей.
- Celery + Redis (T-101).
- AI-генерация аватаров через внешнее API (T-102).
- Email-уведомления (создана сессия / ты приглашён / сессия через 24ч).
- Polish UI: анимации и accessibility (F-015, F-016).

Цель: вовлечение.

## Non-goals

Чтобы не расползалось:

- Мобильное приложение. Фронт — web, generated by Claude Design.
- Платные фичи / подписки.
- Публичная регистрация без invite.
- Поддержка других настольных игр. Если появится запрос — отдельный проект или refactor в Phase 4.
- Real-time (websockets) чат. Чат под матчем — обычный REST + polling / refetch.
